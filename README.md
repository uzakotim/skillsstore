**SkillsStore** is a powerful Retrieval-Augmented Generation (RAG) dashboard designed to transform your PDF library into an interactive knowledge base. Upload books, extract insights, and let AI guide your learning journey with personalized curricula and deep-dive lessons.

---

## ✨ Key Features

- 📁 **Smart PDF Ingestion**: Upload and automatically index PDF books. Text is extracted, chunked, and vectorized for instant retrieval.
- 🔍 **Vector Search Engine**: Perform semantic searches across your entire library or filter by specific books to find the most relevant context.
- 🤖 **AI-Powered Q&A**: Ask questions and receive context-aware answers generated by **Gemma 2** (via Ollama), backed by your own documents.
- 🎓 **Automated Learning Paths**: Generate structured curricula from any book to master complex topics step-by-step.
- 📖 **Deep-Dive Lessons**: Request detailed explanations and examples for specific concepts identified in your learning paths.
- 🛠️ **Developer Debugging**: Built-in tools to inspect the underlying knowledge segments and database state.

## 🚀 Getting Started

### Prerequisites

Before you begin, ensure you have the following installed:

1.  **Node.js & Yarn**: For the frontend and package management.
2.  **Rust**: To build the Tauri backend.
3.  **Ollama**: Running locally on your machine.
    - Pull the required models:
      ```bash
      ollama pull gemma2:2b
      ollama pull nomic-embed-text
      ```
4.  **macOS Specifics**: Install `libomp` to support Faiss (vector engine):
    ```bash
    brew install libomp
    ```

### Installation

1.  **Clone the repository**:
    ```bash
    git clone https://github.com/your-username/skillsstore.git
    cd skillsstore
    ```

2.  **Install dependencies**:
    ```bash
    yarn install
    ```

3.  **Run the application**:
    ```bash
    yarn tauri dev
    ```

## 📖 How it Works

1.  **Processing**: When you upload a PDF, the backend extracts the text using `pdf-extract`.
2.  **Embedding**: The text is split into manageable chunks, and each chunk is converted into a vector representation using `nomic-embed-text`.
3.  **Indexing**: These vectors are stored in a **Faiss** index for lightning-fast similarity searching.
4.  **Retrieval**: When you search or ask a question, the system finds the most relevant "Knowledge Grains" from the index.
5.  **Generation**: The retrieved context is sent to **Gemma 2** to synthesize a coherent, accurate response.

## 🤝 Troubleshooting

### Faiss Build Error (macOS)
If you encounter errors related to `faiss-sys` or OpenMP during build, ensure `libomp` is installed and linked correctly:
```bash
brew install libomp
```

### Database Issues
The app stores data in your system's local data directory. If you need to reset the app, you can delete the `storage` folder within the app's data path.
